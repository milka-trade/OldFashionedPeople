## 📊 3. v36 동작 메커니즘

### 매수 흐름 (Thread 1)
```
매시 정각 → Screener: KRW 200+ 코인 → 5~8개 후보 선정
              ↓
            EMA4HTracker: 후보 코인의 4H EMA 정배열 자격 확인
              ↓  
        BuyEngine.check_buy_signal: 1H BB ≤30%, RSI 25~55, 거래량 충족
              ↓
            execute_buy: 시장가 매수 + sell_engine 자동 등록
```

### 매도 흐름 (Thread 2 — 30초 주기)
```
보유 코인 순회
   ↓
SellEngine.check_sell_signal:
   D+0 (24h 미만): 가속손절 + 당일-3% + 절대-5%만 적용
   D+1 이후: 5조건 평가 (EMA50이탈 / 데드크로스 / 트레일링 / RSI반전 / -5%)
   ↓
execute_sell: 시장가 매도 + sell_engine 자동 제거
```

### 시간 청산 0건 (사용자 핵심 지시 #2,3)
- ❌ 24시간 강제 청산 없음
- ❌ 48시간 강제 청산 없음
- ❌ 16시간 수익 청산 없음
- ❌ 09:00 급락 청산 없음
- ✅ 추세가 살아있으면 며칠이라도 보유 유지

## 🛡️ 5. 안전장치 (LIVE_MODE 진입 후 확인)

### 자동 적용된 안전장치 (코드에 이미 포함)
| 항목 | 위치 | 동작 |
|---|---|---|
| 매수 후 30분 유예 | TrendSellEngine | 매수 직후 30분간은 D+0 손절 미발동 |
| 절대 손절 -5% | TrendSellEngine | D+0/D+1 모두 -5% 도달 시 즉시 매도 |
| 당일 -3% 손절 | TrendSellEngine | D+0 + 매수 30분 후부터 적용 |
| 가속 손절 | TrendSellEngine | 5분봉 3연속 음봉 + RSI<25 |
| buy_price=0 매도 차단 | execute_sell | 동기화 실패 코인 보호 (★ 패치 적용) |
| 매수 차단 시간대 | EMATrendBuyEngine | 08:50~09:30 매수 금지 |
| 일일 매수 5건 제한 | EMATrendBuyEngine | 과도한 매수 방지 |
| 재진입 4시간 쿨다운 | EMATrendBuyEngine | 같은 코인 4시간 내 재매수 금지 |
| 연속 손실 쿨다운 | check_consecutive_losses | 3연속 손실 시 30분 매수 차단 |

## 🎯 9. 첫 24시간 후 확인 포인트

다음 4가지를 디스코드 알림으로 확인하시면 v36이 정상 작동하는 것입니다:

1. ☐ **매시 정각 5~8개의 새 후보 코인 알림**이 정상적으로 옴
2. ☐ **매도 알림 사유에 "시간청산", "24h", "48h" 같은 단어가 없음**
3. ☐ **보유 코인 알림에 EMA 상태(↗50↑ 등)가 표시됨**
4. ☐ **하루 매수 건수가 5건 이하**로 자동 제한됨

## 📞 문제 발생 시

콘솔 로그 + 디스코드 알림 + `holding_coins` 상태를 함께 캡처해 주시면 즉시 분석 가능합니다.

특히 다음 로그가 중요합니다:
- `[BUY SIGNAL]` / `[SELL SIGNAL]` 발화 시점의 사유 텍스트
- `[스크리너]` 실행 결과 (Tier1/Tier2 통과 코인 수)
- `[Sync]` 동기화 결과
